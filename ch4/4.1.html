4.1 Test Techniques Overview

نظرة عامة على تقنيات الاختبار - ISTQB Foundation Level v4.0

Learning Objective: FL-4.1.1 (K2) Distinguish black-box, white-box and experience-based test techniques

1

What are Test Techniques? - إيه هي تقنيات الاختبار؟

"Test techniques support the tester in test analysis (what to test) and in test design (how to test)."

Test Techniques دي الطرق والأساليب اللي بتساعد المختبر في حاجتين أساسيتين:

  • 📋 Test Analysis - تحليل الاختبار: إيه اللي هنختبره؟
  • 🔧 Test Design - تصميم الاختبار: إزاي هنختبره؟
2

Purpose and Goals - الهدف والغاية

"Test techniques help to develop a relatively small, but sufficient, set of test cases in a systematic way."

الهدف الأساسي من Test Techniques إننا نعمل:

🎯 الأهداف الرئيسية:

  • ✅ مجموعة test cases صغيرة العدد (relatively small)
  • ✅ بس كافية ومؤثرة (sufficient)
  • ✅ بطريقة منهجية ومنظمة (systematic way)
  • ✅ مش عشوائية أو بدون تخطيط

🇪🇬 بالمصري: "العبرة بالكيف مش بالكم" - نعمل اختبارات قليلة بس مفيدة ومؤثرة

3

What They Help With - بيساعدوا في إيه؟

"Test techniques also help the tester to define test conditions, identify coverage items, and identify test data during the test analysis and design."

Test Techniques بتساعد المختبر في تحديد 3 حاجات مهمة جداً:

📋 Test Conditions

الشروط والحالات اللي هنختبرها

📊 Coverage Items

العناصر اللي لازم نغطيها في الاختبار

💾 Test Data

البيانات المناسبة واللازمة للاختبار

4

Reference Standard - المرجع الرسمي

"Further information on test techniques and their corresponding measures can be found in the ISO/IEC/IEEE 29119-4 standard."

المعلومات الكاملة عن Test Techniques موجودة في:

📚 المراجع الرسمية:

  • 🏛️ ISO/IEC/IEEE 29119-4 - المعيار الدولي الرسمي
  • 📖 مراجع أكاديمية متعددة من خبراء الاختبار
5

ISTQB Classification - التصنيف الرسمي

"In this syllabus, test techniques are classified as black-box, white-box, and experience-based."

في المنهج ده، Test Techniques مقسمة لـ 3 أنواع رئيسية:

🎯

Black-Box

اختبار من الخارج

⚪

White-Box

اختبار من الداخل

👨‍💼

Experience-Based

بناءً على الخبرة

6

🎯 Black-Box Test Techniques

"Black-box test techniques (also known as specification-based techniques) are based on an analysis of the specified behavior of the test object without reference to its internal structure."

🔍 الخصائص الأساسية:

  • 📛 الاسم البديل: Specification-based techniques
  • 📋 مبنية على: تحليل السلوك المحدد للبرنامج
  • 🚫 بدون الرجوع: للهيكل الداخلي للبرنامج
  • 🇪🇬 بالمصري: زي ما تختبر آلة حاسبة من برة - تدخل 2+3 وتتوقع 5
7

🎯 Black-Box Independence - الاستقلالية

"Therefore, the test cases are independent of how the software is implemented. Consequently, if the implementation changes, but the required behavior stays the same, then the test cases are still useful."

المميزة الكبيرة لـ Black-Box:

✅ Independent

Test Cases مستقلة عن طريقة التنفيذ

✅ Reusable

لو التنفيذ اتغير والسلوك فضل زي ما هو، الاختبارات لسه مفيدة

🇪🇬 مثال: لو غيرت محرك العربية بس العربية لسه بتمشي نفس السرعة، اختباراتك لسه صالحة

8

⚪ White-Box Test Techniques

"White-box test techniques (also known as structure-based techniques) are based on an analysis of the test object's internal structure and processing."

🔍 الخصائص الأساسية:

  • 📛 الاسم البديل: Structure-based techniques
  • 🏗️ مبنية على: تحليل الهيكل الداخلي والمعالجة
  • 👀 تحتاج رؤية: الكود والتصميم الداخلي
  • 🇪🇬 بالمصري: زي ما تفحص محرك العربية من جوة - تشوف كل جزء يشتغل صح
9

⚪ White-Box Dependency - الاعتمادية

"As the test cases are dependent on how the software is designed, they can only be created after the design or implementation of the test object."

الخاصية المهمة لـ White-Box:

⚠️ Dependent

Test Cases معتمدة على تصميم البرنامج

⏰ After Design

مينفعش تتعمل إلا بعد التصميم أو التنفيذ

🇪🇬 يعني: عشان تختبر المحرك من جوة، لازم الأول المحرك يتصنع وتشوف تصميمه

10

👨‍💼 Experience-Based Test Techniques

"Experience-based test techniques effectively use the knowledge and experience of testers for the design and implementation of test cases."

🧠 الخصائص الأساسية:

  • 🎓 مبنية على: معرفة وخبرة المختبر
  • ⚡ استخدام فعال: للمعرفة المتراكمة
  • 🎯 في تصميم وتنفيذ: Test Cases
  • 🇪🇬 بالمصري: زي الدكتور المتمرس اللي يشخص المرض من خبرته
11

👨‍💼 Experience-Based Effectiveness - الفعالية

"The effectiveness of these techniques depends heavily on the tester's skills."

🎯 نقطة مهمة جداً:

⚠️ فعالية Experience-Based معتمدة جداً على:

🎓 المهارات
Tester's Skills
📚 المعرفة
Knowledge
⏳ الخبرة
Experience
12

👨‍💼 Experience-Based as Complementary - مكملة للآخرين

"Experience-based test techniques can detect defects that may be missed using the black-box and white-box test techniques. Hence, experience-based test techniques are complementary to the black-box and white-box test techniques."

🌟 القوة الحقيقية لـ Experience-Based:

💪 المميزات الفريدة:

  • 🔍 تكتشف عيوب: ممكن تفوت على Black-Box و White-Box
  • 🤝 مكملة ومش بديلة: للنوعين التانيين
  • ➕ تضيف قيمة: مش تحل محل الطرق التانية

🇪🇬 المعادلة الصحيحة:
Black-Box + White-Box + Experience-Based = Complete Testing

✅

Final Summary - الخلاصة النهائية

🎯 Key MCQ Points

🎯 Purpose
  • • Support test analysis & test design
  • • Develop small but sufficient test cases
  • • In a systematic way
🎯 Black-Box
  • • = Specification-based
  • • Independent of implementation
  • • Test cases remain useful if behavior same
⚪ White-Box
  • • = Structure-based
  • • Dependent on software design
  • • Created after design/implementation
👨‍💼 Experience-Based
  • • Effectiveness depends on tester's skills
  • • Can detect missed defects
  • • Complementary to others
