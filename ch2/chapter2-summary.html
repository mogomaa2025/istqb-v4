🔤 Keywords & Definitions

📖 Essential Testing Vocabulary

Master these 17 key terms to build a solid foundation in testing throughout the SDLC

acceptance testing
Formal testing with respect to user needs, requirements, and business processes conducted to determine whether or not a system satisfies the acceptance criteria and to enable the user, customers or other authorized entity to determine whether or not to accept the system.
Test Level
black-box testing
Testing, either functional or non-functional, without reference to the internal structure of the component or system.
Test Type
component integration testing
Testing performed to expose defects in the interfaces and interaction between integrated components.
Test Level
component testing
The testing of individual software components. Also known as unit testing, module testing, or program testing.
Test Level
confirmation testing
Testing that runs test cases that failed the last time they were run, in order to verify the success of corrective actions.
Change-Related Testing
functional testing
Testing conducted to evaluate the compliance of a component or system with functional requirements.
Test Type
integration testing
Testing performed to expose defects in the interfaces and in the interactions between integrated components or systems.
Test Level
maintenance testing
Testing the changes to an operational system or the impact of a changed environment to an operational system.
Test Context
non-functional testing
Testing the attributes of a component or system that do not relate to functionality, e.g., reliability, efficiency, usability, maintainability and portability.
Test Type
regression testing
Testing of a previously tested program following modification to ensure that defects have not been introduced or uncovered in unchanged areas of the software, as a result of the changes made.
Change-Related Testing
shift-left
An approach to testing where testing is performed earlier in the SDLC, moving testing activities to the left on the project timeline.
Testing Approach
system integration testing
Testing the integration of systems and packages; testing interfaces to external organizations (e.g., Electronic Data Interchange, Internet).
Test Level
system testing
Testing an integrated system to verify that it meets specified requirements.
Test Level
test level
A group of test activities that are organized and managed together. A test level is linked to the responsibilities in a project.
Test Organization
test object
The component or system to be tested.
Test Element
test type
A group of test activities aimed at testing specific characteristics of a software system, or a part of a system, based on specific test objectives.
Test Classification
white-box testing
Testing based on an analysis of the internal structure of the component or system.

📊 Chapter 2 Summary

🎯 Testing Throughout the Software Development Lifecycle - Complete Overview

Chapter 2 provides comprehensive coverage of testing in different contexts, from SDLC integration to test levels, types, and maintenance. This chapter establishes the framework for applying testing principles across various development methodologies and operational scenarios.

🔄

2.1 Testing in SDLC Context

SDLC Impact & Practices: How different development models affect testing approach, good practices for all SDLCs, test-first approaches (TDD, ATDD, BDD), DevOps integration, shift-left strategy, and retrospectives for continuous improvement.

📊

2.2 Test Levels & Types

Structure & Classification: Five test levels (Component, Component Integration, System, System Integration, Acceptance), four test types (Functional, Non-functional, Black-box, White-box), confirmation vs regression testing, and ISO 25010 quality characteristics.

🔧

2.3 Maintenance Testing

Production Context: Testing changes in operational systems, three maintenance categories (Corrective, Adaptive, Perfective), scope factors (Risk, System Size, Change Size), triggers (Modifications, Upgrades, Retirement), and impact analysis.
